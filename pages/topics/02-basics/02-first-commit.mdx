# Pierwszy *komit*

<Callout type="info" emoji="ℹ️">
    *Commit* można porównać do Crl + s - zapisania wersji projektu. Robimy to żeby ustawić sobie taki "checkpoint".\
    Później będziemy mogli przywrócić starszą wersję projektu lub porównać swoje "za-commitowane" zmiany.
</Callout>

Żeby mieć co zapisać, musimy coś zrobić. Stwórz zwykły plik tekstowy, np `test.txt` i napisz w nim co tylko Ci się podoba.

Ponieważ commit to trochę więcej niż zwykły zapis projektu, musimy trochę więcej się narobić. Zacznijmy od wybrania plików, które będą zawarte w commit-cie.

```bash
  git add <nazwa_towjego_pliku>
```
