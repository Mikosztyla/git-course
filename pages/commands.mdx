## Lista najważniejszych komend *Git-owych*:


### **git init**
Tworzy nowe repozytorium Git w bieżącym katalogu.
   ```bash
   git init
   ```

---

### **git clone**
Kopiuje zdalne repozytorium na lokalny komputer.
   ```bash
   git clone <URL> [<nazwa_katalogu>]
   ```
   - **`--depth <n>`**: Pobiera tylko ostatnie `<n>` commitów, co przyspiesza klonowanie (np. przy dużych repozytoriach).
   - **`--branch <branch_name>`**: Klonuje konkretną gałąź.

---

### **git add**
Dodaje pliki do obszaru staging (do kolejnego commitu).
   ```bash
   git add <plik>          # Dodaje pojedynczy plik
   git add .               # Dodaje wszystkie zmienione pliki
   git add -A              # Dodaje wszystkie pliki (zależnie od wersji Git może być tożsamy z `git add .`)
   ```

---

### **git commit**
Tworzy nowy commit z plikami dodanymi do staging.
   ```bash
   git commit -m "<wiadomość>"
   ```
   - **`-m "<wiadomość>"`**: Dodaje wiadomość opisującą commit (krótką i opisową).
   - **`-a`**: Automatycznie dodaje wszystkie zmienione i śledzone pliki do commitu.
   - **`--amend`**: Zmienia ostatni commit (używane np. do poprawy wiadomości lub dodania plików).

---

### **git status**
Pokazuje aktualny status repozytorium: zmodyfikowane, usunięte pliki oraz pliki w staging.
   ```bash
   git status
   ```

---

### **git push**
Wysyła lokalne commity do zdalnego repozytorium.
   ```bash
   git push <zdalne_repo> <gałąź>
   ```
   - **`-u <remote> <branch>`**: Ustawia domyślny zdalny dla gałęzi (pierwsze `push` wymaga tej flagi, kolejne mogą być `git push`).
   - **`--force` lub `-f`**: Wymusza `push` (stosowane ostrożnie, ponieważ zmiany w zdalnym repozytorium).

<Callout type="error" emoji="️🚫">
    `push --force` **nadpisze** zdalne repozytorium lokalną wersją gałęzi.\
    Po wykonaniu dostaniesz pytanie `Czy na pewno chcesz to zrobić?`, więc tak, dość niebezpieczna operacja.
</Callout>

---

### **git pull**
Pobiera i łączy zmiany ze zdalnego repozytorium z lokalną gałęzią.
   ```bash
   git pull <zdalne_repo> <gałąź>
   ```
   - **`--rebase`**: Używa `rebase` zamiast `merge`, co pozwala na liniową historię zmian.

---

### **git fetch**
Pobiera zmiany ze zdalnego repozytorium, ale ich nie łączy. Przydatne do sprawdzenia, jakie zmiany zaszły w zdalnym repo.
   ```bash
   git fetch <zdalne_repo>
   ```

---

### **git branch**
Zarządza gałęziami w repozytorium.
   ```bash
   git branch               # Wyświetla listę lokalnych gałęzi
   git branch <nazwa>       # Tworzy nową gałąź
   git branch -d <nazwa>    # Usuwa gałąź (jeśli jest już zmergowana)
   ```
   - **`-D <branch_name>`**: Wymusza usunięcie gałęzi, nawet jeśli nie została zmergowana.

---

### **git checkout**
Przełącza się między gałęziami lub przywraca pliki do określonego stanu.
   ```bash
   git checkout <gałąź>
   git checkout <plik>      # Przywraca plik do ostatniego commitu
   ```
   - **`-b <nowa_gałąź>`**: Tworzy nową gałąź i od razu się na nią przełącza.

---

### **git merge**
Łączy zmiany z jednej gałęzi do bieżącej gałęzi.
   ```bash
   git merge <gałąź>
   ```
   - **`--no-ff`**: Tworzy merge commit nawet w przypadku, gdy możliwe jest szybkie połączenie (fast-forward).
   - **`--abort`**: Anuluje merge, jeśli wystąpiły konflikty.

---

### **git rebase**
Przenosi zmiany na bieżącej gałęzi na szczyt innej gałęzi. Przydatne do tworzenia bardziej liniowej historii.
   ```bash
   git rebase <gałąź>
   ```
   - **`--interactive` lub `-i`**: Pozwala na interaktywne przenoszenie commitów (re-ordering, zmiany w historii).
   - **`--continue`**: Kontynuuje rebase po rozwiązaniu konfliktów.
   - **`--abort`**: Anuluje rebase i przywraca stan sprzed jego rozpoczęcia.

---

### **git reset**
Przestawia bieżący `HEAD` na wybrany commit.
   ```bash
   git reset <commit>
   ```
   - **`--soft`**: Przenosi `HEAD` na wybrany commit, ale zostawia zmiany w staging.
   - **`--mixed`**: Przenosi `HEAD` na wybrany commit i usuwa zmiany ze staging, ale pozostawia je w katalogu roboczym.
   - **`--hard`**: Przenosi `HEAD` na wybrany commit, całkowicie usuwając zmiany ze staging i katalogu roboczego.

---

### **git log**
Wyświetla historię commitów.
   ```bash
   git log
   ```
   - **`--oneline`**: Wyświetla uproszczoną wersję logu, gdzie każdy commit to jedna linia.
   - **`--graph`**: Wyświetla historię w formie graficznego drzewa.
   - **`-p`**: Pokazuje zmiany (diff) dla każdego commitu.

<Callout type="info" emoji="ℹ️">
    Wciśnij `q`, żeby przerwać wyświetlanie
</Callout>

---

### **git stash**
Zapisuje bieżące zmiany w pamięci tymczasowej, umożliwiając tymczasowe przeniesienie się na czystą gałąź.
   ```bash
   git stash              # Zapisuje bieżące zmiany w stash
   git stash pop          # Przywraca ostatni stash i usuwa go
   git stash list         # Wyświetla listę staszy
   ```
